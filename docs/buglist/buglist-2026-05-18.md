# Buglist 2026-05-18

## 13:58 — tools/mod.rs: execute_tool_calls 拦截被禁止工具时未发送 SSE 事件

- **模块**: `crates/core/src/tools/mod.rs`
- **现象**: BDD 测试 "Plan Agent 禁止调用 bash 工具" 失败，断言期望收到 `ToolError` 事件，但事件列表为空 `[]`。
- **根因**: `execute_tool_calls` 在检测到工具不被当前 Agent 允许时，直接 `return Part::ToolError { ... }`，**没有**通过 `on_tool_event` 回调发送 SSE `Part` 事件。因此客户端（包括 BDD 测试 harness）无法感知到拦截动作。
- **修复方案**: 在拦截分支中，先构造 `Part::ToolError`，然后通过 `cb.lock()` 获取回调并发送 `SseEvent::Part { part: error_part.clone() }`，再返回该 `error_part`。
- **相关 Commit**: `d440fe6` feat(tests): add BDD tests for Agent type tool filtering

## 13:54 — tests/bdd/mod.rs + tui_flow_e2e.rs: 未处理 AgentInfo SSE 事件导致编译失败

- **模块**: `tests/bdd/mod.rs`, `tests/e2e/tui_flow_e2e.rs`
- **现象**: `cargo build` 失败，报错 `pattern &SseEvent::AgentInfo { .. } not covered`。
- **根因**: Task 7 在 `SseEvent` 中新增了 `AgentInfo` 变体，但 BDD 和 E2E 测试 harness 中的 match 表达式未添加对应分支。
- **修复方案**: 在两个文件的 SSE 事件 match 中各添加 `Ev::AgentInfo { .. } => SseEvent { event_type: "AgentInfo".to_string(), ... }` 分支。
- **相关 Commit**: `3450a7a` fix(tests): handle AgentInfo SSE event in E2E/BDD test harness
