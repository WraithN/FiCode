**重构动机**：
1. 引入外部可观测后端 Langfuse，让 LLM 调用、工具调用、压缩等全链路可在 Langfuse UI 上 trace 化展示
2. 避免业务代码同时维护 `TurnLogger` + 新 OTel 调用两套可观测代码，从源头杜绝双写不一致
3. 本地 `spans.jsonl` 作为兜底，Langfuse 不可达时不丢数据；启动期 daemon 识别 pending 行
4. 对接 Langfuse 的 OTLP HTTP 端点，避免引入 `tonic`/gRPC 这类重依赖

**重要安全 / 正确性修复**（review 阶段发现并即时修复）：
1. `LocalJsonlBridge` 重开文件 + `.expect()` panic 风险 → 改为 `LocalJsonlExporter::export_batch(&self)` 直调，消除双文件 handle 与 O_APPEND interleaving 风险
2. `install()` 不幂等（OnceLock 静默 no-op）→ 入口加 PROVIDER.get().is_some() 提前返回
3. `shutdown()` 未重置 `ENABLED` → 补 `ENABLED.store(false)`
4. `messages_snapshot` 直接 set_attribute（可 100KB+ 且含敏感数据） → 走 `redact_and_truncate`
5. `model_name="unknown"` / `provider_name="unknown"` 占位 → 给 `AIClient` trait 加 `model_name()` + `provider_kind()`，OpenAI/Anthropic/Mock 三个实现各自返回真实值
6. attrs.rs 中 4 个 fi_code 常量字符串值与 plan 不一致 → 按 plan 对齐（`fi_code.messages_snapshot` 等）
7. spec 中 `public_key`/`secret_key` snake_case 与项目 camelCase 约定不一致 → 改 spec + 代码 `#[serde(rename = "publicKey"/"secretKey")]`
